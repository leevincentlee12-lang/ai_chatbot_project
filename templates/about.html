Project Overview

An educational platform for guided algebra learning.

This project is designed to support algebra study through structured explanations, practice problems, answer checking, lessons, and progress tracking. It focuses on explainable learning behaviour rather than a black-box conversation model.

Current Focus

  1. Classify algebra learning requests.
  2. Route work to the correct engine.
  3. Generate structured educational responses.
  4. Track practice progress across a browser session.

Purpose

The platform helps students practise algebra in a focused environment. It can solve supported equations, explain methods, validate algebraic steps, generate practice questions, and show progress indicators based on real activity inside the app.

The project is strongest in algebra workflows such as linear equations, quadratics, factorising, simplifying expressions, simultaneous equations, hints, step validation, and generated practice.

Educational Goals

The system is built around clarity, feedback, and measurable progress. It aims to help a learner understand both the answer and the method.

Structured Explanations

Responses are presented as answer, method, why, check, and next step sections.

Adaptive Practice

Practice questions use the existing progression system to adjust difficulty.

Learning Evidence

Progress pages show attempts, correct answers, accuracy, skill scores, and recent activity.

Technologies Used

The stack is intentionally simple so the project is easy to run, explain, and maintain.

  • Python and Flask for routes, APIs, and server-side templates
  • SymPy for algebra parsing, simplification, factoring, and solving
  • SQLite-backed state storage for anonymous learning progress
  • HTML, CSS, and JavaScript for the responsive frontend interface
  • GitHub for version control and Render for public deployment

Deployment Details

The public version is deployed as a Flask web service connected to the GitHub repository.

  • Render builds and runs the Flask app from the repository
  • GitHub Desktop is used to commit and push changes
  • The free Render instance may sleep after inactivity
  • Current storage is suitable for a demonstration, not long-term production records

System Architecture

The project is separated into routing, classification, engines, validation, progression, formatting, and data modules. This keeps the app easier to extend without rewriting the whole codebase.

A student question moves through routing, subject-specific handling, solving, structured feedback, and progress updates.

app.py

Flask routes, JSON endpoints, pages, sessions, and request handling.

core/

Routing support, deterministic classification, parsing, validation, progression, and state storage.

engine/

Subject engines and algebra workflows, including solving, practice, lessons, and answer checking.

data/

Lesson content, constants, and intent definitions used by the classifier.

utils/

Shared formatting helpers for readable educational responses.

templates/ and static/

Responsive pages, styling, and browser-side interaction logic.

Future Improvements

The next useful improvements should make the platform more reliable, measurable, and classroom-ready rather than adding unnecessary features.

  • Add user accounts if long-term individual progress needs to be preserved
  • Move persistent learning data to a hosted database for production deployment
  • Expand test coverage around intent routing and algebra edge cases
  • Add teacher-facing summaries only after the student data model is stable
  • Broaden the lesson set while keeping algebra as the clearest strength
